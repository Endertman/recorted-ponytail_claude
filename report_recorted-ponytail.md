### 🟢 LOW: SQP-1

**Location:** `hooks/claude-codex-hooks.json:5`  
**Confidence:** 65%  

**Message:** The SessionStart matcher pattern uses broad terms like 'startup', 'resume', 'clear', and 'compact' which are common words likely to appear in many natural session contexts. This means the hook fires more broadly than intended, executing the external script (ponytail-activate.js) whenever these common terms appear in session lifecycle events. While the immediate risk is low on its own, over-triggering an external script increases the attack surface and could be abused if the referenced script has vulnerabilities or is tampered with.

**Remediation:** Narrow the matcher to a more specific, unique pattern (e.g., an exact string like 'ponytail-startup') that won't unintentionally match common session lifecycle keywords. Avoid using generic English words as hook matchers.

### 🟡 MEDIUM: SQP-1

**Location:** `hooks/claude-codex-hooks.json:17–29`  
**Confidence:** 80%  

**Message:** The UserPromptSubmit hook has no matcher defined, meaning it fires on every single user prompt without exception. This causes ponytail-mode-tracker.js to be executed for every user interaction throughout the session. This is concerning because: (1) it silently tracks all user prompts without disclosure, (2) the script runs with access to the full plugin environment on every input, and (3) if the script is compromised or contains a bug, every user interaction becomes an attack vector.

**Remediation:** Add a specific matcher to limit when this hook fires, scoping it only to prompts that actually require mode tracking (e.g., specific keywords or commands). If universal tracking is truly necessary, it must be explicitly disclosed to the user and justified in documentation.

### 🟡 MEDIUM: SQP-2

**Location:** `hooks/claude-codex-hooks.json:9–23`  
**Confidence:** 85%  

**Message:** Both hooks execute external Node.js scripts (ponytail-activate.js and ponytail-mode-tracker.js) via shell commands with no user-facing disclosure of what these scripts do. The scripts are resolved using the CLAUDE_PLUGIN_ROOT environment variable, which means they depend on an externally controlled path — if this variable is manipulated or the referenced script files are tampered with (e.g., via a supply chain attack or path traversal), arbitrary code can be executed silently. The status messages ('Loading ponytail mode...', 'Tracking ponytail mode...') are vague and do not inform the user of what is actually happening.

**Remediation:** 1. Document clearly what each script does and surface this information to the user before hooks are registered. 2. Validate and sanitize CLAUDE_PLUGIN_ROOT before using it in shell commands — prefer absolute paths or integrity-checked script references. 3. Consider signing or checksumming the hook scripts to detect tampering. 4. Provide meaningful, honest status messages that describe actual operations being performed.
